# 윈도우 실험 실행 가이드

## 1. 최초 환경 셋업

### 필요한 것
- Python 3.10+ (3.14 권장)
- Ollama + `gemma4:e4b` 모델
- 이 프로젝트의 `experiments/` 폴더 전체

### 폴더 복사

맥에서 윈도우로 `experiments/` 폴더 전체를 복사한다:

```
experiments/
├── config.py
├── schema.py
├── orchestrator.py
├── system_prompt.py
├── run_experiment.py
├── measure.py
├── tasks/
│   └── taskset.json
└── results/          ← 결과가 여기에 저장됨
```

### 가상환경 + 의존성

```powershell
cd experiments
python -m venv .venv
.venv\Scripts\activate
pip install httpx
```

### Ollama 확인

```powershell
ollama list
# gemma4:e4b가 보여야 함

# 없으면:
ollama pull gemma4:e4b
```

### config.py 확인

윈도우의 Ollama 주소가 다르면 수정:
```python
# 기본값은 localhost:11434 — 보통 윈도우에서도 동일
OLLAMA_BASE_URL = "http://localhost:11434"
```

## 2. 실험 실행 순서

```
실험 1 → 실험 2 → 실험 3 (+ 실험 4는 추후)
```

### 실험 1: Assertion 상한

```powershell
python run_experiment.py assertion-cap
# 예상 소요: 2-4시간
```

### 실험 2: 다단계 루프

```powershell
# (선택) 실험 1 결과 분석 후 config.py의 ASSERTION_SOFT_CAP 조정
python run_experiment.py multiloop
# 예상 소요: 6-12시간 → 밤에 돌려놓기 권장
```

### 실험 3: 오류 전파

```powershell
python run_experiment.py error-propagation
# 예상 소요: 3-5시간
```

## 3. 결과 분석 (윈도우에서 바로 가능)

```powershell
# 실험 1 분석
python measure.py results/exp01_assertion_cap_*.json

# 실험 2 분석
python measure.py results/exp02_multiloop_*.json

# 마크다운 보고서 생성
python measure.py results/exp02_multiloop_*.json --save-md results/exp02_report.md
```

## 4. 결과 파일을 맥으로 반입

### 가져올 파일

실험 완료 후 `experiments/results/` 안에 생성된 **JSON 파일만** 가져오면 된다:

```
results/
├── exp01_assertion_cap_20260409_HHMMSS.json      ← 이것
├── exp01_assertion_cap_partial_*.json             ← 중간 저장 (optional)
├── exp02_multiloop_20260409_HHMMSS.json           ← 이것
├── exp03_error_propagation_20260409_HHMMSS.json   ← 이것
└── (기타 partial 파일들)
```

### 맥에서의 저장 위치

```bash
# 프로젝트 루트 기준
experiments/results/  ← JSON 결과 파일을 여기에 놓는다
```

이미 있는 exp00 결과 파일들과 같은 위치다:
```
experiments/results/
├── exp00_baseline_20260408_102018.json    ← 기존
├── exp01_assertion_cap_XXXXXXXX.json      ← 새로 가져옴
├── exp02_multiloop_XXXXXXXX.json          ← 새로 가져옴
└── exp03_error_propagation_XXXXXXXX.json  ← 새로 가져옴
```

### 반입 후 분석

맥에서 분석 + 결과 문서 업데이트:

```bash
cd experiments

# 분석 + 마크다운 보고서 자동 생성
.venv/bin/python3 measure.py results/exp01_assertion_cap_*.json \
  --save-md ../docs/reference/results/exp-01-assertion-cap.md

.venv/bin/python3 measure.py results/exp02_multiloop_*.json \
  --save-md ../docs/reference/results/exp-02-multiloop-quality.md

.venv/bin/python3 measure.py results/exp03_error_propagation_*.json \
  --save-md ../docs/reference/results/exp-03-error-propagation.md
```

## 5. 트러블슈팅

### Ollama 타임아웃

`config.py`의 `OLLAMA_TIMEOUT`이 600초로 설정되어 있다.
E4B 모델 첫 로딩 시 시간이 걸릴 수 있으므로, 첫 실행 전에 한 번 워밍업:

```powershell
ollama run gemma4:e4b "Hello"
# 모델 로딩 완료 후 Ctrl+D로 나간다
```

### 빈 응답 / JSON 파싱 실패

콘솔에 `⚠ Error: JSON parse failed`가 반복되면:
- 모델이 JSON 대신 자연어로 응답하는 경우
- `orchestrator.py`의 `num_predict`를 4096으로 올려볼 것 (현재 2048)

### 메모리 부족

E4B는 약 10GB VRAM 필요. VRAM이 부족하면 CPU 모드로 전환되어 매우 느려진다.
`ollama ps`로 GPU 사용 여부 확인.
