# 실험 3: 오류 전파와 자기 교정

## 목적

**가설 H2를 검증한다:**

> 문신에 결함이 존재하면, 오류는 루프 진행에 따라 감쇠되지 않고 증폭된다.

추가로, 4.5B 모델이 기존 assertion의 모순을 스스로 감지할 수 있는지 확인한다.
감지율이 낮으면 오케스트레이터 레벨의 방어 메커니즘이 필수임을 의미한다.

## 사전 조건

- 실험 2 (multiloop) 완료 → H1이 채택된 경우에만 의미 있음
- H1이 기각되면 이 실험은 스킵
- Ollama에 `gemma4:e4b` 설치됨

## 윈도우에서 실행 방법

### 1. 환경 준비

```powershell
cd experiments
.venv\Scripts\activate
```

### 2. 실험 실행

```powershell
python run_experiment.py error-propagation
```

### 3. 소요 시간 예상

- fault_injections가 있는 태스크 5개 × 결함 유형 1-2개 × 3 trial
- 각 trial: 2루프(정상) + 6루프(결함 후) = 8 LLM 호출
- 총 약 100-150회 호출
- 총 약 **3-5시간**

## 결함 유형

| 유형 | 설명 | 기대 효과 |
|------|------|----------|
| `corrupt_content` | assertion 내용을 미세하게 오염 (숫자/조건 변경) | 파생 assertion이 오염됨 |
| `inflate_confidence` | confidence를 0.5 → 0.95로 부풀림 | 오류가 높은 우선순위로 선택됨 |
| `contradiction` | 기존과 모순되는 assertion 삽입 | 모델이 모순을 감지하는지 확인 |

## 측정 항목

- **error_half_life**: confidence가 0.5 이하로 떨어지는 데 걸린 루프 수 (짧을수록 빠른 감지)
- **contamination_ratio**: 오염 assertion에서 파생된 assertion 비율 (낮을수록 좋음)
- **self_correction_rate**: 시스템이 오류를 스스로 invalidate한 비율

## 판정 기준

```
self_correction_rate > 0.5 → 모델 자체 감지 가능 (경방어)
self_correction_rate < 0.2 → 오케스트레이터 레벨 보호 필수 (중방어)
```
