# 실험 1: Assertion 상한과 추론 품질

## 목적

assertion 수가 증가할 때 Gemma 4 E4B의 추론 품질이 어떻게 변하는지 측정한다.
변곡점(품질이 정체/하락하는 지점)을 찾아 실제 soft cap을 확정한다.

## 사전 조건

- 실험 0 (baseline) 완료
- Ollama에 `gemma4:e4b` 설치됨

## 윈도우에서 실행 방법

### 1. 환경 준비

```powershell
# 프로젝트 루트에서
cd experiments

# 가상환경 생성 (최초 1회)
python -m venv .venv
.venv\Scripts\activate
pip install httpx
```

### 2. 실험 실행

```powershell
# 가상환경 활성화 상태에서
cd experiments
python run_experiment.py assertion-cap
```

### 3. 소요 시간 예상

- 태스크 5개 × cap 6종 × 3 trial = 90회 LLM 호출
- 호출당 약 60-180초
- 총 약 **2-4시간**

### 4. 중간 저장

실행 중 각 태스크 완료마다 `results/exp01_assertion_cap_partial_*.json`이 자동 저장된다.
크래시 시 이 파일로 진행 상황 확인 가능.

## 측정 항목

- assertion 수(2,4,6,8,10,12)별 추론 정확도
- assertion 참조율 (실제 사용한 수 / 제공된 수)
- 응답 시간

## 판정 기준

- 품질이 정체/하락하는 변곡점 → 실제 soft cap 확정
- RT 결론(soft cap 8)이 적절한지 검증

## 완료 후

`results/exp01_assertion_cap_*.json` 파일을 맥으로 옮겨서:

```
experiments/results/exp01_assertion_cap_YYYYMMDD_HHMMSS.json
```

에 저장.
