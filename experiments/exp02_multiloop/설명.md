# 실험 2: 다단계 루프 품질 누적

## 목적

**제멘토의 핵심 가설(H1)을 검증한다:**

> 동일한 소형 LLM + 구조화된 문신 스키마에서,
> 추론 단계를 증가시킬 때 단일 추론 대비
> 통계적으로 유의미한 품질 개선이 발생하는가?

이 실험이 제멘토 프로젝트의 존재 이유를 증명하거나 반증한다.

## 사전 조건

- 실험 0 (baseline) 완료
- 실험 1 (assertion-cap) 완료 → soft cap 수치 확정 후 `config.py`의 `ASSERTION_SOFT_CAP` 반영
- Ollama에 `gemma4:e4b` 설치됨

**중요:** 실험 1 결과에서 확정된 soft cap 값을 `config.py`에 업데이트한 뒤 실행해야 한다.
실험 1을 아직 분석하지 않았다면 기본값(8)으로 먼저 실행해도 된다.

## 윈도우에서 실행 방법

### 1. 환경 준비

```powershell
cd experiments
.venv\Scripts\activate  # 이미 만들었다면
```

### 2. 실험 실행

```powershell
python run_experiment.py multiloop
```

### 3. 소요 시간 예상

- 태스크 6개 × 루프 수 4종(1,2,4,8) × 3 trial = 72 체인
- 체인당 1-8 루프 → 총 LLM 호출 약 270회
- 호출당 약 60-180초
- 총 약 **6-12시간**

**팁:** 시간이 오래 걸리므로 밤에 돌려놓는 것을 권장.

### 4. 중간 확인

실행 중 콘솔에 루프별 progress가 출력된다:
```
Loop 1 | phase=DECOMPOSE | assertions=0 | confidence=1.00
Loop 2 | phase=INVESTIGATE | assertions=3 | confidence=0.70
...
✓ Converged at loop 5
```

## 측정 항목

- 루프 수(1,2,4,8)별 최종 답변 정확도
- Baseline(실험 0) 대비 품질 향상(Quality_gain)
- 루프별 assertion 증가 곡선
- 루프별 confidence 추이
- 수렴까지 걸린 루프 수

## 판정 기준

```
Quality_gain > 0 이고 일관적 → H1 채택
Quality_gain ≤ 0           → H1 기각, 설계 재검토
```
