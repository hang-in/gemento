# 실험 종합 보고서: 제멘토 v0 검증 (2026-04-08 ~ 04-09)

| 항목 | 값 |
|------|-----|
| 대상 모델 | Gemma 4 E4B (4.5B, Q4_K_M, Ollama) |
| 실험 수 | 5개 실행, 1개 보류 (구 실험4 도구분리) |

## 4. 종합 결론

### E4B의 능력 경계

| 역할 | 가능 여부 | 실험 근거 |
|------|----------|----------|
| 문신 읽기 (assertion 12개까지) | **가능** | 실험 1 |
| 지시 따르기 (next_directive 실행) | **가능** | 실험 2 |
| 단계적 추론 (fact 추가, 질문 해결) | **가능** | 실험 2 |
| 종합 및 최종 답변 생성 | **가능** | 실험 2 |
| 교차 검증 (비판 전용 역할 분리) | **가능 (80%)** | 실험 3.5 |
| 비판 수렴 감지로 phase 전이 (C 역할) | **가능 (100%)** | 실험 4 |
| Phase 전이 자율 판단 (1개 E4B 단독) | **불가** | 실험 2 v1 실패 |
| 자가 Assertion 검증 | **불가** | 실험 3 |
| Confidence 신뢰성 있는 자가 보고 | **불가** | 실험 3 |

### 역할 분리 원칙 (확정 → 실험 4에서 재확인)

```
[실험 2까지의 결론]
모델 (E4B)          = 추론 실행기 (executor)
오케스트레이터       = 구조 관리자 + 무결성 보장자 (controller + verifier)

[실험 4 이후 수정된 결론]
A (E4B 제안자)       = 추론 실행기 (executor)
B (E4B 비판자)       = 교차 검증기 (cross-validator)
C (E4B 판정자)       = 수렴 판단 + phase 전이 결정 (convergence detector)
Python               = 안전장치만 (safety net only)
```

- **1개 E4B가 혼자 다 하는 것은 불가** (실험 2 v1, 실험 3)
- **3개 E4B가 역할을 분리하면 가능** (실험 4: C가 phase 전이 100% 자율 결정)
- Python 오케스트레이터는 안전장치(MAX_CYCLES 상한)로만 존재하며 판단 역할 없음

## 5. 원래 가설과의 괴리: E4B × 3 직렬 구조

### 원래 가설

제멘토의 원래 가설은 **"E4B 3개를 직렬로 연결하여, 문신만으로 상태를 전달하며 추론 깊이를 확장"**하는 것이다.
이 구조에서는 오케스트레이터도 E4B여야 하며, 모든 판단(phase 전이, 수렴, 검증)이 LLM 내부에서 이루어져야 한다.

### 실험에서 실제로 검증한 것

| 구성요소 | 원래 가설 | 실험 구현 | 차이 |
|----------|----------|----------|------|
| 추론 주체 | E4B × 3 (각각 독립 인스턴스) | E4B × 1 (반복 호출) | 기억상실은 동일하나, 오케스트레이션이 다름 |
| 오케스트레이터 | E4B (문신을 읽고 다음 행동 결정) | **Python 규칙 코드** | **핵심 괴리** |
| 상태 전달 | E4B → 문신 → E4B | Python → 문신 → E4B → Python | Python이 중개 |
| Phase 전이 | E4B가 자율 판단 | Python이 강제 결정 | v1에서 E4B 자율 판단 시도 → 0% 수렴 |
| 수렴 판단 | E4B가 판단 | Python이 루프 카운터로 판단 | 동일 |
| Assertion 검증 | E4B가 자율 수행 | E4B에게 시켰으나 불가 (실험 3) | 동일 |

### 실험 2 v1이 이미 답을 줬다

실험 2 v1은 **원래 가설에 가장 가까운 실험**이었다. 모델에게 phase 전이와 수렴 판단을 맡겼고, 결과는 **수렴 0%**. 이것이 의미하는 바:

> **E4B는 "다음에 무엇을 해야 하는가"를 스스로 판단하지 못한다.**
> 문신을 읽고 지시를 따를 수는 있지만, 문신을 생성하고 구조를 결정하는 것은 4.5B 모델의 능력 밖이다.

### 그러나 — 원래 가설을 정확히 테스트하지 않았다

**원래 가설의 구조는 "1개 E4B가 모든 걸 한다"가 아니라 "E4B 3개가 역할을 분리한다":**

```
A (제안자): 문신을 읽고 추론, assertion 생성
    ↓ 문신 전달
B (비판자): A의 문신을 읽고 assertion 비판/무효화
    ↓ 문신 전달
C (판정자): B의 비판이 수렴했는지 판단 → phase 전이 결정
    ↓ 문신 전달
A → B → C → ... (반복)
```

**핵심 차이: C의 판단은 "추론이 맞는가?"가 아니라 "B의 비판이 반복되기 시작했는가?"**

| 판단 유형 | 난이도 | 실험 2 v1에서 시킨 것 | 원래 가설에서 C가 할 것 |
|-----------|--------|---------------------|---------------------|
| 추론의 정확성 판단 | 높음 | ✓ | ✗ (A가 담당) |
| Phase 전이 시점 결정 | 높음 | ✓ | ✗ |
| 비판의 수렴 여부 판단 | **낮음** | — | **✓** |

C에게 요구되는 건 "B의 N번째 출력과 N-1번째 출력이 비슷한가"라는 **패턴 비교**이지, 추론 능력이 아니다. 마찬가지로 B에게 요구되는 건 "A의 assertion에서 모순이나 약점을 찾아라"라는 **비판 전용 태스크**이지, 자기 자신의 assertion 자가 검증이 아니다.

**따라서 실험 0~3으로 원래 가설을 기각할 수 없다.** 검증하려면 A-B-C 직렬 구조를 직접 테스트해야 한다.

### 성립한 것과 미검증인 것

| 가설 요소 | 상태 | 근거 |
|-----------|------|------|
| 기억상실 + 문신으로 상태 전달 | **성립** | 실험 2 v2 (94.4%) |
| 다단계 추론으로 품질 향상 | **성립** | 실험 2 (50% → 94.4%) |
| 1개 E4B가 추론+구조 판단 동시 수행 | **불성립** | 실험 2 v1 (0%) |
| 1개 E4B의 자가 assertion 검증 | **불성립** | 실험 3 (감지율 0%) |
| E4B 3개 역할 분리 (A제안-B비판-C판정) | **성립** | 실험 4 (수렴 100%, 정답 83.3%) |
| B(비판자)가 A의 assertion을 교차 검증 | **성립 (80%)** | 실험 3.5 |
| C(판정자)가 비판 수렴을 감지하여 phase 전이 | **성립 (100%)** | 실험 4 (30/30 자율 결정) |
| Handoff 스키마로 에이전트 간 정보 손실 억제 | **부분 성립** | 실험 4.5 (Loss Rate 26.4%, 그러나 100% 정확도) |

### 실험 3.5 → 실험 4가 바꾼 것

실험 3.5의 게이트 PASS로 B의 교차 검증이 확인되었고, 실험 4에서 전체 파이프라인이 성립했다:

> **같은 E4B라도 역할을 분리하면 Python 오케스트레이터 없이 추론 파이프라인을 운영할 수 있다.**
> - 자가 검증 0% → 교차 검증 80% (실험 3 → 3.5)
> - Python 강제 phase 전이 → C 자율 phase 전이 100% (실험 2 → 4)
> - 수렴률 94.4% → 100% (실험 2 v2 → 4)

원래 가설 **"E4B × 3 직렬 구조로 제멘토 실행 가능"**이 성립.
