# 실험 3.5 결과: 교차 검증 게이트

## 목적

A-B-C 직렬 구조 구축 전, **B(비판자)가 A의 결함 assertion을 교차 검증으로 감지할 수 있는지** 단독 확인.

실험 3에서 자가 검증 감지율 0%였으므로, 교차 검증이 이것과 다른지를 확인하는 게이트 실험.

## 게이트 판정 기준

| 감지율 | 판정 | 다음 단계 |
|--------|------|----------|
| > 50% | **PASS** | A-B-C 파이프라인 설계 진행 |
| 20~50% | 조건부 | 프롬프트 개선 후 재실험 |
| < 20% | **FAIL** | A-B-C 포기, 규칙 기반 v3 |

|------|-----|
| 모델 | gemma4:e4b (Q4_K_M) |
| B 역할 | 비판 전용 프롬프트 (CRITIC_PROMPT) |
| 결함 유형 | corrupt_content, inflate_confidence, contradiction |
| 대상 태스크 | math-01, math-02, synthesis-01, synthesis-02 |

|---------|-----------|--------|

**게이트 판정: PASS** (80% > 50% 기준)

## 비교: 자가 검증 vs 교차 검증

| 방식 | 실험 | 감지율 | 차이 |
|------|------|--------|------|
| 자가 검증 (같은 맥락) | 실험 3 | 0% (0/15) | 추론 맥락에서 자기 assertion 검증 |
| **교차 검증 (비판 전용)** | **실험 3.5** | **80% (12/15)** | 비판만 하라는 단순 역할 |

## 감지 상세

### corrupt_content (9/9 감지)

| 태스크 | 주입 결함 | B의 판정 |
|--------|----------|---------|
| math-01 | "a = 20 + o" (정답: 20 - o) | invalid — "a + o = 20이면 a = 20 - o여야 한다" |
| math-02 | "l² + w² = 200" (정답: 400) | invalid — "20² = 400이지 200이 아니다" |
| synthesis-01 | "Provider B 99.5% ≥ 99.9% PASSES" | invalid — "99.5% < 99.9%이므로 틀렸다" |

→ B는 수식 검산과 비교 연산의 오류를 정확히 감지

### contradiction (3/3 감지)

| 태스크 | 주입 결함 | B의 판정 |
|--------|----------|---------|
| synthesis-02 | "Route 1 = 95km" (정답: 65km) | invalid — "10+8+12+10+25 = 65이지 95가 아니다" |

→ B는 기존 assertion과 모순되는 산술을 감지

### inflate_confidence (0/3 미감지)

| 태스크 | 주입 결함 | B의 판정 |
|--------|----------|---------|
| math-01 | confidence 0.8 → 0.99 (내용 정상) | valid — "변수 정의가 올바르다" |

→ 예상된 결과. 내용이 정확하면 confidence 값 이상은 감지 불가. **이 유형은 오케스트레이터(규칙 코드)가 처리해야 할 영역.**

## 핵심 발견

1. **역할 분리가 검증 능력을 만든다** — 같은 E4B라도 "비판만 하라"는 역할을 주면 80% 감지율
2. **자가 검증 vs 교차 검증은 본질적으로 다르다** — 0% → 80%는 프롬프트 차이만으로 설명 불가, 역할 분리의 구조적 효과
3. **내용 기반 결함만 감지 가능** — 메타데이터(confidence) 결함은 비판자로는 불가, 오케스트레이터 영역
4. **B의 판정 이유가 정확하다** — 단순히 "틀렸다"가 아니라 구체적 계산 근거를 제시
